# Design Decisions: YNAB-Inspired Redesign

**Date:** 2026-03-22  
**Author:** Design Director (design-cd)  
**Status:** Implemented

---

## 1. Layout Architecture

### Decision: Two-panel master-detail layout
**Rationale:** YNAB's power comes from the budget table being the centrepiece, with context appearing beside it — not on separate pages. The current card-grid layout forces scanning across isolated containers. A table + context panel puts everything in one scan.

**Implementation:**
- **Left panel (60%):** Budget table with collapsible groups, inline editing
- **Right panel (40%):** Context panel showing either selected category detail or monthly overview
- Main content area uses `flex` with fixed proportions inside the existing sidebar layout

### Decision: Keep the sidebar navigation
**Rationale:** The existing sidebar is well-implemented and provides clear navigation. No reason to discard it.

### Decision: TopBar becomes month-aware
**Rationale:** YNAB's topbar is always month-contextual. "Ready to Assign" is the single most important number in a budget-first workflow.

**New TopBar elements:**
- `< March 2026 >` month navigator with prev/next
- "Ready to Assign: £X" — income minus total budgeted
- Sync status with manual trigger

## 2. Budget Table Design

### Decision: Table rows, not cards
**Rationale:** Cards create visual noise. Each card has its own border, padding, shadow — multiplied by 20+ categories, it's overwhelming. Table rows are dense, scannable, and information-rich per pixel.

**Row anatomy:**
```
[▼/▶] [icon] Category Name    [████████░░░] £340 spent  £160 left  [⋯]
```

### Decision: Collapsible group headers with roll-up totals
**Rationale:** YNAB groups categories. The current `BudgetGroupCard` does this but wastes space. A group header row with chevron toggle + aggregate totals is more efficient.

**Group header anatomy:**
```
▼ 🏠 FIXED BILLS                              £1,200 / £1,500    £300 left
```

### Decision: Colour-coded available amounts
**Rationale:** YNAB uses green/yellow/red on the "Available" column, not on cards. This is faster to scan.

- **Green (mint):** Under 80% spent
- **Yellow:** 80-100% spent
- **Red (coral):** Over budget

### Decision: Sinking funds as a table section, not separate cards
**Rationale:** Sinking funds are budgets. Treating them as a separate card grid breaks the mental model. They become a collapsible group row in the budget table with their own progress indicators.

## 3. Inline Editing

### Decision: Replace ALL budget/category Dialog modals with inline editing
**Rationale:** The current flow is: click → modal opens → fill form → submit → modal closes. This is 4 interactions for a 1-interaction task. YNAB just lets you click a number and type.

**Inline edit component (`InlineEdit`):**
- Click to activate → field becomes an input
- `Enter` or blur → save (calls mutation)
- `Escape` → cancel
- Shows loading spinner during save
- Two variants: `text` and `currency`

**Applied to:**
- Budget amounts (click the number → type new amount)
- Category names (click the name → edit inline)
- Group names (click the group header → edit inline)

### Decision: Keep Dialog for destructive actions only
**Rationale:** Delete confirmations should be deliberate. Alert dialogs stay for delete operations.

---

## 4. Context Panel (Right Panel)

### Decision: Dual-mode right panel
**Rationale:** When nothing is selected, show a useful overview. When a category is selected, show its detail.

**Default state (nothing selected):**
- Monthly spend summary stat
- Spending trend sparkline
- Top categories list

**Selected category state:**
- Category name + status badge
- 6-period spend sparkline (using Recharts)
- Budget amount (inline editable)
- Transaction list for this category this period
- Each transaction row has inline category reassignment

---

## 5. Transaction Page Redesign

### Decision: Date-grouped full-width table, inline category editing
**Rationale:** Transactions grouped by date (TODAY, YESTERDAY, MON 18 MARCH) is more scannable than a flat paginated list. Inline category editing removes the modal.

**Changes:**
- Remove `Card` wrapper — full-width table
- Date group headers with day totals
- Click category pill on any row → inline dropdown for reassignment
- Keep month navigation, search, and category filters

### Decision: Running balance column (optional, shown when space permits)
**Rationale:** YNAB shows this. It's useful for spotting anomalies.

---

## 6. Transactions in Context Panel

### Decision: Category transactions load in right panel, not a separate page
**Rationale:** Clicking a budget row should show that category's transactions immediately, without navigation. This is the YNAB interaction model.

**Implementation:** The context panel fetches transactions filtered by the selected category and current budget period using the existing `useTransactions` hook with category filter.

---

## 7. Mobile Responsiveness

### Decision: Stack panels vertically on mobile
**Rationale:** Two-panel layout doesn't work below 1024px. Below that breakpoint:
- Budget table goes full-width
- Context panel becomes a bottom drawer/sheet triggered by selecting a row
- TopBar collapses month nav to a compact strip

---

## 8. What Was Kept vs Changed

### Kept:
- Sidebar navigation (with existing nav items)
- All API hooks (`useApi.ts`) — zero changes
- All API types (`api.ts`) — zero changes
- Design tokens (colours, fonts, Tailwind config)
- shadcn/ui base components (Button, Input, Label, Select, AlertDialog)
- Account selector in TopBar area
- Recharts for charting

### Changed:
- `dashboard.tsx` — complete rewrite (two-panel layout)
- `transactions.tsx` — rewrite (date groups, inline editing)
- `budgets.tsx` — simplified (now redirects to dashboard or serves as import-only page)
- `TopBar` component — replaced with `TopBarBudget` variant
- `BudgetGroupCard` → deprecated, replaced by `BudgetGroupHeader` + `BudgetTableRow`
- `BudgetBar` → deprecated (kept for backwards compat but not used in new layout)
- `SinkingFundCard` → deprecated, sinking funds rendered as `BudgetTableRow` with pot metadata
- `TransactionRow` → kept but enhanced with inline category dropdown

### New Components:
- `InlineEdit` — reusable inline-editable field (text + currency variants)
- `BudgetTable` — full YNAB-style budget table with groups
- `BudgetGroupHeader` — collapsible group header row
- `BudgetTableRow` — individual budget line item row
- `ContextPanel` — right-side detail/overview panel
- `TopBarBudget` — month-aware topbar with "Ready to Assign"
- `SpendSparkline` — mini 6-period sparkline
- `TransactionRowV2` — enhanced transaction row with inline category dropdown
- `CategoryDropdown` — inline category selector dropdown

## 10. Performance Considerations

- **Virtualisation:** Not needed yet (typical budget = 20-30 rows). If it grows past 100, add `react-window`.
- **Debounced inline edits:** `InlineEdit` debounces API calls on blur, not on every keystroke.
- **Optimistic updates:** Category reassignment uses optimistic update via TanStack Query's `onMutate`.
- **Lazy right panel:** Context panel only fetches transactions when a category is selected.

---

## 11. Accessibility

- **Keyboard navigation:** Tab through budget rows, Enter to select, Escape to deselect
- **ARIA:** Budget table uses `role="table"`, rows use `role="row"`, group headers use `role="rowgroup"`
- **Focus management:** InlineEdit traps focus when active, returns focus on save/cancel
- **Screen readers:** Status colours backed by text labels ("Over budget", "On track", "Warning")
